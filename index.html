The Age of Async Agents — Cognition 如何用 Devin 重新定义软件开发

嘉宾：Walden Yan · Cognition 联合创始人 & CPO / Cole Murray · OpenInspect 创建者

这期 Latent Space 是 Swyx 与 Cognition 联合创始人 Walden Yan 和 OpenInspect 创建者 Cole Murray 的深度对话。Cognition 刚刚宣布以 260 亿美元估值完成 10 亿美金融资，而 Devin 在公司内部已经贡献了 80% 的代码提交——从去年的 16% 到现在的 80%。

AI coding 的三个阶段。 第一阶段是 Copilot 式的自动补全——开发者还是主角，AI 只是加速器。第二阶段是本地 agent——Claude Code、Cursor 的 agent 模式，你在终端里看着它干活。第三阶段就是现在的异步背景 agent——你给它一个 spec，它在云端独立完成从理解需求到提交 PR 的整个流程。Walden 坦诚地指出，2025 年 12 月的模型能力跃升是关键转折点——在那之前 "spec 到 PR" 不实用，之后 Devin 的合并 PR 数量增长了 7 倍。

"大脑"与"机器"分离。 Devin 架构的核心原则是把推理决策层和执行环境分开。每个 agent 拿到一个完整的 VM 快照——不是 Docker 容器——里面已经配好了 repo、依赖、scoped secrets。Agent 干完活，VM 就销毁。这样做的好处是安全性和权限控制：agent 只获得完成任务所需的最小权限。

Repo setup 是最大瓶颈。 Walden 认为 agent 采用的最大障碍不是模型能力，而是"你能不能在一个干净的环境里把这个项目跑起来"。很多公司的开发环境依赖于工程师本地积累的隐性知识——环境变量、私有包、特殊构建步骤。Agent 没有这种隐性知识。所以 Cognition 卖的不只是 Devin，而是一整套企业级基础设施。

视频验证与 "永远在线" agent。 测试比 computer use 难得多——Agent 会启动应用、录屏、用视觉模型验证 UI 是否正确。Cognition 内部已经有常驻 agent，永远盯着特定 feature area 的 issue，自动分诊、自动生成修复 PR。PM 在 Slack 里发一条消息描述需求，Devin 直接生成 PR——模糊了 "谁能写代码" 的边界。

Vibe coding 的失控与人类新角色。 Walden 对此很诚实：纯自动合并大约两周后就会崩溃——代码库会退化到最差工程师的水平。他们用 AI 代码审查、lint 规则、Semgrep 来检测 "agent 代码坏味道"。人类的角色从 "写代码" 变成了 "审代码" 和 "定规则"。Cole Murray 从开源角度验证了同样的趋势——他的 OpenInspect 框架让中小团队也能搭建异步 coding agent 系统。
