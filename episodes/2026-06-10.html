Story 01

WWDC 2026：苹果发布史上最大 Siri 升级，但底层是谷歌的 Gemini

六月九号，苹果在 WWDC 2026 发布了十多年来 Siri 最大的一次升级——Siri AI。新 Siri 跨所有平台运行，能执行复杂的连续操作，对话更自然、更有上下文，而且不再把问题转交给 ChatGPT 这类第三方。但最微妙的地方是：它真正的大脑，是谷歌的 Gemini。苹果自己的模型没跟上，于是做了一个务实的决定——界面、入口、调度、隐私这一层自己牢牢握住，生成答案的模型则租谷歌的。对用户来说，Siri 从哪个模型出来根本不重要，重要的是它就在 Dynamic Island 上、在你下滑的界面里、在每一台 iPhone 里。iOS 27 还把 Apple Intelligence 铺进了 Safari、信息、密码、快捷指令、家庭。苹果赌的是——能把模型无缝塞进十亿台设备日常操作里的能力，比模型本身更稀缺。开发者测试版已放出，正式版今年秋天上线，支持 iPhone 16 全系、iPhone 15 Pro 及以上。

📎 Apple Newsroom · TechCrunch

Story 02

NotebookLM 大升级：从研究工具变成能交活的工作台

六月八号的一次更新，让 NotebookLM 换了一个物种。以前它的输出是笔记和音频，现在它能直接生成 PDF 报告、Word、Excel、PPT、图表，带数据可视化，从面板下载、生成完还能编辑。更底层的两个变化：一是加了 Deep Research，搜索不再局限于你上传的资料，能往外找新来源；二是每个笔记本都配了一台安全的云端电脑，能自己写代码、跑代码做复杂分析，整个东西跑在 Gemini 3.5 和 Antigravity 上。它从一个帮你理解资料的工具，变成了一个帮你完成工作的工作台。信号很清晰：AI 产品的下一步，不是回答得更好，而是交付得更完整。

📎 Google Blog · @joshwoodward

Story 03

Claude Code 满一周年：从提示到调度，交互范式正在变

Claude Code 满一年，创造者 Boris Cherny 六月八号发长推复盘。几个数字：他已经八个月没手写过一行代码，有些日子同时调度好几万个 Agent，大部分工作在手机上、睡觉前开始。整个 Anthropic 今年代码产出涨了八倍，人均工程效率提升近 70%，人数翻了三倍。关键的范式变化——他现在主要用 auto mode 而非 plan mode（一年前 Agent 三十秒就跑偏，现在能连跑五到二十小时），还用 routines 让 Claude 自己持续推进、在他发现之前就修好 bug。Swyx 引用的新基准 FrontierCode 也印证：去年底模型可靠性跨过临界点，人就敢往上爬一层抽象——从写代码，变成写让 Agent 自己干活的循环。

📎 Fortune · @bcherny

Story 04

把三件事连起来：模型在商品化，价值在往调度层和交付层迁移

苹果把模型外包给谷歌、自己守住入口；NotebookLM 不比谁答得聪明、比谁交付得完整；Boris 不再写代码、而是调度 Agent。三件事都在说同一句话：模型本身不再是壁垒。一个足够好的模型谁都买得到、租得到，真正的价值在往两头跑——一头是调度层，谁能把一群 Agent 编排好、让它们可靠地连续工作；另一头是交付层，谁能把 AI 的能力变成用户真正能用的成品和入口。呼应 Aaron Levie 这周的观点：再多智能也替代不了上下文。你独有的上下文、你对用户的理解、你调度 AI 的方法论——这些没法被一个更大的模型直接买走。你的护城河，从来都不在那个你租来的模型里。

📎 @levie

Building an AI Guardian for Enterprise with Onyx Security CEO Maxim Bar Kogan

当一个人一天要调度上万个 Agent，谁来盯着这些 Agent？Onyx Security 的答案是：训练一批"很小、很笨、但只擅长一件事"的模型——判断某个 Agent 动作要不要叫一个更聪明的模型来复核。Maxim 用下棋打比方：高手大多数走子靠直觉，只在关键局面停下来深算。把这套"直觉 + 关键时刻深算"做进监督层，才能在成本和延迟都可控的前提下，看住那些比我们更聪明的 Agent。对做产品的人，这是"调度层"之上又长出的一层——治理层。
