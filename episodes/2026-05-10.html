🎧 AI & I by Every · 2026-05-08

Claude 平台的秘密：Anthropic 产品 + 工程负责人首次公开设计哲学

嘉宾：Angela（平台产品负责人）· Caitlin（平台工程负责人）· 约 43 分钟

这一期 Dan Shipper 请来了 Anthropic 平台团队最核心的两个人——一个管产品、一个管工程——深度拆解了 Cloud Managed Agents 的设计逻辑、企业 agent 的真实落地案例、以及未来一年的平台愿景。

真正卡住你的不是 prompt，是基础设施。 Caitlin 指出，大多数开发者以为难点在 harness engineering（调 prompt、优化上下文窗口），但实际上真正的墙是基础设施——长时间稳定运行、断连恢复、安全沙箱、对话历史存储。Anthropic 内部自己搭了太多次，于是决定做一次做对，开放给所有人。

路径依赖比你想象的严重。 Angela 揭示了一个深层问题：你选择让模型怎么工作——文件系统、skills、还是其他方式——会锁定模型未来的发展路径。不同的 harness 设计在评估测试中表现差异巨大。Anthropic 因此在 Managed Agents 里做了强倾向性的设计选择，因为他们相信这是发挥 Claude 最大潜力的方式。

"通用 harness + 换模型"的时代结束了。 Angela 坦诚回答了模型锁定的担忧——以前那种超级通用的 harness 随意换模型的方式已经不适用了。每个模型正在走向特化路径，最佳实践是把 harness 和模型当作一个整体来优化。

Managed Agents 搭 Managed Agents。 最有趣的企业案例是 Anthropic 内部的法务审核 agent——市场团队提交文案，agent 先做一轮审核，能过的直接通过，不确定的再转人工。用户修改 agent 行为时不是改代码，而是跟另一层 Claude 对话，底下多层 Claude 互相协作完成修改。

未来：结果 + 预算，其余 Claude 搞定。 Angela 描述的终极愿景是——用户只需要提供想要的结果和预算，模型选择、agent 架构、sub-agent 拆分全部由 Claude 自动完成。Claude 正在变得越来越"理解自己"，最终实现自我编排。Caitlin 则务实地补充——在那个世界里，平台扩展性才是最大的挑战。

Notion 产品负责人 Max Schoening：AI 时代，Agency 比技能更重要

嘉宾：Max Schoening · Notion 产品负责人（前 Google PM / GitHub 设计 VP）

Vibe coding 让人人都能写代码了，但大量产出的是更多的软件，而不是更好的软件。真正拉开差距的是 agency——独立判断和推动事情发生的能力。他的"tiny core"理论：伟大的产品都有一个极简核心——iPhone 的多点触控、GitHub 的 pull request、Notion 的 block。每个项目的前 10% 现在是"免费"的，真正的价值在后面的 90%。
