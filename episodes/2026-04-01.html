01 · 头条

苹果正面怼 Vibe Coding：App Store 审核体系遭遇 AI 原生冲击

苹果本周对 vibe coding 应用出手——Replit、Vibecode 被封禁更新，「Anything」直接从 App Store 下架。理由是违反 2.5.2 条款：不允许应用在审核通过后改变核心功能。 这背后是一场产品哲学的正面碰撞：苹果的平台治理建立在「静态审核」假设上，但 AI 原生应用是动态的、有状态的，每次运行结果都不同。审核队列已从24小时扩展到一周，传统人工审核体系被 AI 生成应用的洪水冲垮。 出路可能是：给成熟开发者设快速通道，或对 AI 生成内容做专门沙箱——但这意味着苹果要翻整个地基，而不只是打补丁。

Cursor 的十字路口：$2B ARR，护城河在哪？

《财富》杂志深度报道《the rapid rise, and very uncertain future, of a $30 billion AI startup》直指核心：Cursor 超过70%的推理跑在 Claude 上，而 Anthropic 既是供应商，也是最直接的竞争对手。 Cursor CEO 的答案是更快的迭代——每五小时更新一个模型 checkpoint，自建 RL。但 Claude Code 三月加了电脑控制、自动模式，Anthropic 直接向用户收费。依赖基础模型的应用，护城河只有两条路：数据飞轮，或极致的分发与体验。Cursor 两条路都在走，但问号还在。

全自动工程部门：General Intelligence 发布 co-founder CTO

纽约 General Intelligence Company 在 Data Driven NYC 发布「co-founder CTO」——全云端、无本地开发、全自动工程部门。新任务触发后，系统在云上起预览环境、调用 Claude Code 写代码、起浏览器 agent 做 QA，通过即自动合并，部署失败自动修复。 内部数据：平均每工程师每天25个 PR 合并；AI tokens 开支有时超过员工薪资。这是一个成本结构的根本转变——资本正在从人力转向算力。

04 · 昨日大事

Anthropic 把 Claude Code 源码泄出去了：KAIROS、潜行模式、反蒸馏工具全曝光

3月31日，Anthropic 在发布 Claude Code v2.1.88 时意外打包了内部 source map 文件，暴露约 2000 个 TypeScript 文件、51 万行代码。安全研究员 Chaofan Shou 第一时间在 X 上公开，帖子数小时内几百万浏览，代码被 fork 超过 4 万次，随后 Anthropic 发出 DMCA 下架请求。 泄露内容中最受关注的有三点：KAIROS 是一个未发布的后台自主 agent 模式，每5分钟心跳询问 Claude "有什么值得做的吗"，可在无人操作时自主修复 bug；潜行模式在贡献开源代码时自动抹去 Anthropic 内部代号；反蒸馏工具会向竞争对手的爬虫注入假工具定义投毒训练数据。此外还有一条让人忍俊不禁的发现——用正则表达式检测用户的挫败感（匹配词：wtf、omfg……），Anthropic 造了最先进的语言模型，却用最原始的 regex 侦测骂声。 Claude Code 创始工程师 Boris Cherny 在 X 直接回应：「这是人工错误，不是工具链的 bug。我们的发布流程有几个手动步骤，其中一步没有正确执行。出错时重要的是认识到，这从来不是某一个人的责任，是流程、文化、或者基础设施的问题。」团队已发布修复版本并补充了自动化校验步骤。 更棘手的是：泄露当天 axios npm 包同时遭供应链攻击，3月31日特定时段内安装或更新 Claude Code 的用户，可能拿到了含远程访问木马的版本。

05

Aaron Levie 的清醒：Agent 效率的上限不是算力，是治理

Box CEO Aaron Levie 一针见血：「对 agents 提升效率的终极限制，不在于技术，而在于安全、合规、治理。企业不可能让 agents 在任何时候对任何数据做任何事。我们一直活在某种幻觉里。」 过去48小时正是印证——苹果封禁 vibe coding 应用，各种合规问题集中爆发。Microsoft Agent 365（每月15美元）作为企业所有 AI agent 的「控制平面」同步推出，Perplexity Computer for Enterprise 接入 Slack 和 Snowflake。产品机会正在从「agent 能做什么」转向「谁来管 agent」。
