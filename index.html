Story 01

OpenAI 收购 Ona：让 Codex 的 agent 跑进你自己的云

六月十一号，OpenAI 宣布收购 Ona——就是原来那家叫 Gitpod 的德国云端开发平台。整个团队会并进 OpenAI 的 Codex。这笔收购的核心，不是又添一个会写代码的模型，而是补上一块"地盘"：Ona 擅长的，是安全的云端执行和编排——让 agent 能在客户自己的云环境里、长时间、持续地跑活儿，代码和数据不出企业家门。OpenAI 给出的数字很猛：现在每周有超过五百万人用 Codex，比今年早些时候涨了百分之四百。潜台词很清楚——光有一个聪明的编程模型已经不够，谁能让这个模型在企业真实环境里安全、持久地连续工作，谁才握住了新的护城河。Ona 的核心成员 Thibault Sottiaux 已在 X 上官宣加入 OpenAI，swyx 等一众建造者转发祝贺。交易尚待监管审查，财务条款未披露。

📎 OpenAI · CNBC · Bloomberg

Story 02

Replit 统一画布：从想法到上架，全在一块画布上

Replit 这周在推它的"统一画布"。Amjad Masad 演示：一块画布上同时放着你的网页应用、手机应用、营销素材、还有上架 App Store 要用的图文材料，点进任何一个都能改——等于把"做一个产品"从头到尾需要的所有东西塞进一个界面，彻底打破"先写代码、再做设计、再做营销"的旧流水线。他还专门发了一条讲成本：Agent 团队把 Fable 模型的使用成本做得"能让胃接受了"，因为 agent 出错少了，里外算下来反而更便宜。这个细节很关键——vibecoding 能不能成为真正的生产工具，瓶颈往往不是聪不聪明，而是出错率和成本。Amjad 另一条说得更直接：第一次完全没挫败感地 vibecoding，进了心流，灵感反而不够用了。这意味着这个形态过了临界点：从玩具变成了工作台。

📎 @amasad

Story 03

Vercel + Shopify：v0 + Cursor 搭的定制店铺，两分钟 500 单

Vercel 的 CEO Guillermo Rauch 这周晒了一个 Vercel + Shopify 项目：一个完全定制的店铺前端，用 v0 加 Cursor 搭出来，跑在 Next.js 上，两分钟内处理了五百多笔订单。两分钟五百单是营销噱头，但真正的信号在"完全定制"四个字。过去想要一个高转化、又长成自己样子的店铺前端，要么花大价钱找团队，要么忍受模板的千篇一律；现在 v0 加 Cursor 让"定制"的成本塌了下来——你可以既用 Shopify 的支付和库存这套后端基础设施，又拥有一个完全属于你自己的前端皮。又是同一条主线的变体：后端基础能力商品化、谁都能接，价值跑到了"你能不能快速搭出一个独一无二的交付层"。后端是租来的，前端的体验和速度才是你的。

📎 @rauchg

Story 04

Snowflake CoWork / CoCo：把 agent 跑进你被治理的数据

把镜头拉到企业级。这个月初的 Snowflake Summit 上发了两个 agent：CoWork 是给普通知识工作者的私人 agent，不用会写 SQL 就能问数据、生成报告；CoCo 是给开发者的数据原生编程 agent。两个底层都是 Claude，Snowflake 跟 Anthropic 签了笔两亿美元的多年合作。又是"agent 跑在你的数据上"——前面是跑在你的云里、你的画布里，这里是跑在被治理好的企业数据上。这次 Summit 被很多人总结成一句话："你的数据，才是真正的 AI 护城河。" 它跟今天的主线是一体两面：模型是商品，但你独有的数据、上下文、对自己业务的理解，没法被一个更大的模型直接买走。把四条连起来——OpenAI 把 agent 塞进你的云，Replit 塞进创作者的工作流，Vercel 塞进交付层，Snowflake 塞进企业数据。模型本身不再是壁垒。2026 年别再问"你用哪个模型"，要问"你的 agent 在谁的地盘上跑、喂的是谁的上下文"。

📎 Snowflake · @levie
