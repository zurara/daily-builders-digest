Railway: Agent-Native Cloud — 35 人团队如何用裸金属数据中心重新定义云基础设施

嘉宾：Jake Cooper · Railway 创始人 / 前 Bloomberg & Uber 工程师

这期 Latent Space 是 Swyx 和 Alessio 与 Railway 创始人 Jake Cooper 的深度对话。Jake 的初衷很简单——把东西部署到生产环境的门槛应该趋近于零。但他做的事情远不止简化部署。

自建裸金属数据中心，3 个月回本，毛利率 70%。 Railway 不是在 AWS 上包一层皮。Jake 把大部分工作负载迁移到了自己的裸金属服务器——投资回收期只有 3 个月，远低于传统云厂商。70% 的利润率用来资助弹性扩展——需要爆发时 burst 到 GCP 或 AWS。这是一种混合策略，但底层的控制权在自己手里。

Agent-Native 意味着完全不同的交互范式。 AI agent 部署软件的方式和人类完全不同——即时启动、动态伸缩、短生命周期的环境。传统云是为人类设计的：你登录控制台、配置实例、部署代码。但当 agent 在一秒钟内要启动一百个微服务时，整个基础设施需要重新设计。Jake 甚至提到了"PR 的死亡"——当 agent 可以直接部署和回滚代码时，pull request 审核流程反而成了瓶颈。

35 个人，300 万用户，每周新增 10 万注册。 这个团队密度惊人。更有意思的是，他们每月 AI coding agent 开支超过 20 万美元——不是在卖 AI，而是在用 AI 构建自己的产品。Railway 的公司花费本身就是 agent-native 开发模式的最佳注解。

5 月 19 日的 GCP 宕机给这个故事添了一笔现实主义注脚——Railway 有多可用区、多区域的 mesh 网络和 HA 光纤互连，但还是受到影响。这恰恰验证了为什么他们要加速拥有自己的硬件——依赖别人的云，就是接受别人的风险边界。
