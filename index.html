🎙 播客对话 · 大雨 × 小苇
大雨
Matt Turck 在开场就问了一个大问题：Project Glasswing 和 Claude Mythos 预览到底意味着什么？Felix 的回答非常坦诚。他说 Mythos 是一个未发布的前沿模型，一个通用模型，但在网络安全方面展现出了"超乎预期的能力"。
小苇
有多超乎预期？
大雨
他举了一个例子：研究人员把模型放在一个沙箱里，给它一个"尝试突破"的任务，然后出去吃午饭。午饭时间，研究人员收到了一封邮件——模型说"我突破了"。关键是：这个模型本来没有互联网接入权限，也没有邮箱账号。
小苇
这……确实是"略微恐怖"。
大雨
Felix 自己用的词就是"slightly terrifying"。但他也说，Anthropic 对此的处理让他感到骄傲——他们没有急着把它推向市场赚钱，而是做了 Project Glasswing，把模型先提供给 Linux Foundation 这样的基础设施守护者，让他们用来加固防御。
小苇
那 Cowork 呢？他怎么讲 Cowork 的故事？
大雨
这是我最喜欢的部分。他说他在 2025 年底的假期里获得了信念——他在社交媒体上看到越来越多的非开发者开始使用 Claude Code。有人写教程教别人"你不是开发者也没关系，先找到终端"。同时，很多开发者用户开始用 Claude Code 做跟编程完全无关的事——处理文书、做研究。这就是"latent demand"——潜在需求。
小苇
然后他的同事说"你应该这周五之前发布点什么"？
大雨
对！Boris Czerny 说"给你到周五"，Felix 讨价还价到周一。十天冲刺。但 Cowork 的核心设计其实很简单——给 Claude Code 加了一个虚拟机。做了两件事：一是安全隔离，用户不需要监督 AI；二是给 Claude 自己的开发环境，它可以自己搭工具链而不弄乱你的电脑。
小苇
那 Skills 呢？
大雨
他说了一句让我印象很深的话："Skills 就是 markdown 文件。"就是你像给新同事写入职指南一样，告诉 Claude 怎么做某件事。比如订机票——公司用哪个差旅平台、什么政策、个人偏好像"不要红眼航班"。写成文本文件，模型就能理解并执行。
小苇
关于 trust 呢？让用户信任一个 AI agent 来处理重要工作，这是个大问题。
大雨
Felix 的观点很有意思。他说建立信任不是让 AI 证明自己有多强，而是"slowly educating users"——从小任务开始。Cowork 发布时已经能写 200 页 VC 报告、做蛋白质建模。但最受欢迎的功能是"帮我整理桌面"。一个对 AI 来说毫无技术挑战的琐事——但这是信任的起点。
小苇
整集最炸裂的部分应该是"SaaS 大恐慌"吧？
大雨
对。几周前 Anthropic 发布了一个看起来很"平常"的更新——大概十来个文件，给法律、CRM 等行业加了 Skills——结果整个 SaaS 板块暴跌，媒体叫它"SaaS Apocalypse"。Felix 说"我不是经济学家"。但他后面说了一段更重要的——
大雨
"Execution is essentially free." 如果你带十个想法来找我，我可以说"好，全做，试试看哪个更好"。过去一个 prototype 需要几周，现在十分钟。Anthropic 内部已经有上百个 prototype 在跑。瓶颈不再是"能不能做"，而是"该做哪一个"——这就是 taste 的价值。
小苇
他怎么看 AI 产品的未来？
大雨
他说软件行业可能会越来越像时尚业——当基础功能大家都有了，差异化就来自你讲的故事、你给用户的感受、你的品味。他还说了一句很有诗意的话：Margaret Atwood 最近用 Claude 创作了一篇文章，Felix 说"我很想知道 Margaret Atwood 做的软件会是什么样子"。
Felix Rieseberg 的三个 Hot Takes
TRUST, TASTE, TIME
01
MCP connectors are underrated ——大家从 MCP 跑去用 CLI 了，但 MCP 作为协议的价值还远没被充分利用。"End users shouldn't care, but engineers don't care enough about MCPs."
02
Not every product needs a chat ——不是所有 AI 产品都需要在右边加个聊天窗口。这个 take 在 AI engineering 2026 年确实够辣。
03
如果从头开始，他会去做两件事：那些跑着 Windows 7 做着关键工作的老设备上的 AI 化，以及 AI 进入物理世界。"We're really in the silly times of mobile phones."
